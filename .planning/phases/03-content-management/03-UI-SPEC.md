# Phase 3 — UI Design Contract

> Visual and interaction contract for the Content Management phase (recipe form, image upload, AI generation, export/import, Anthropic API key in Settings). Generated by gsd-ui-researcher. All values are locked to the existing dark-amber bourbon design system in `app/css/app.css` — no new tokens are introduced.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla CSS custom properties on `:root` in `app/css/app.css`) |
| Preset | not applicable — bespoke design system established in Phase 2 |
| Component library | none — IIFE view modules render raw HTML strings |
| Icon library | inline SVG + Unicode glyphs (✓, ✕, ⏳, ✨, ←, +). No icon library |
| Font | `'Georgia', serif` site-wide; `monospace` for recipe IDs and amounts |

**Hard constraints (from CLAUDE.md):**
- No framework, no bundler, no npm runtime dependencies for CSS/JS
- Single stylesheet: `app/css/app.css`. All new styles append to this file
- IIFE view module pattern: `const FooView = (() => { ... return { render }; })();`
- ZIP packing: `JSZip` is the only allowable third-party JS dependency for this phase, loaded from CDN via `<script src="...">` in `app/index.html`. No npm install. No build step.

## Interaction & State Contract

### Recipe Form (renderForm)

- **Mode:** in-place content replacement inside `#main-content`. NOT a route. NOT a modal.
- **Title:** "New Recipe" when `isNew`, "Edit: {name}" when editing. 1.3rem amber.
- **Back button:** `.back-btn` (line 568). Label "← Back to Recipes" (new) or "← Back to Recipe" (edit). Returns to grid or detail respectively.
- **Required fields visual cue:** label suffixed with `*` (already at recipes.js:450, 459). No red star; the `*` inherits label color `var(--text-dim)`.
- **Save gate (D-02):** if `name`, `creator`, at least one `ingredients[].name`, or `method` is empty, show `Utils.toast(..., 'error')`. Do NOT inline-highlight fields red.
- **Saving state:** Save button disabled, label changes to "Saving…" (existing pattern at recipes.js:610).
- **Success:** green toast "Recipe created." / "Recipe updated.", navigate to detail view.

### AI Prompt Field (D-11–D-14)

- **Visibility:** rendered ONLY when `isNew === true`. Removed from DOM (not just hidden) on edit.
- **No-key state:** if `localStorage.getItem('bb_anthropic_key')` is falsy, render the prompt wrap but disable the Generate button with `:disabled` styling (`opacity: 0.4`, line 301) and tooltip `title="Add your Anthropic API key in Settings to use AI generation"`. The prompt textarea remains enabled (user can type but cannot submit).
- **In-flight state:** Generate button shows the 14px `.spinner` + label "Generating…". All other form fields disabled (programmatic). Cancel button stays enabled.
- **Success:** form fields populate inline with the response. Toast: "AI draft loaded — review and save." (info / blue).
- **Error:** keep form intact, re-enable Generate. Red toast: "Generation failed: {message}".

### Image Upload (D-04–D-06)

**Already implemented** in recipes.js:343-419. No design changes required. The existing pattern (Choose File → filename appears → Upload to GitHub button → status text) is locked as-is. Status colors: `--text-muted` (idle), `--green` (success), `--red` (error). Filename pattern `{id}_{timestamp}.{ext}` per D-05.

### Export / Import (D-07–D-10)

- **Export section** lives inside existing `#sect-export` in `settings.js:148-159`. Update button labels:
  - "Export All Data (JSON)" → `Export All Data (ZIP)` (D-07)
  - "Export AI Context (text)" → `Export for AI (text)` (EXPORT-02)
- **Drop zone** rendered inside `#st-import-area`. Single `<div class="import-drop-zone">` containing:
  - Line 1 (0.95rem, text-dim): `Drop a ZIP export file here`
  - Line 2 (0.82rem, text-muted): `— or —`
  - Line 3: `.btn .btn-secondary .btn-sm` labeled `Choose File…`
- **Drag-over visual:** add `.dragover` class. Border becomes solid amber, background becomes amber-tint `rgba(212,148,58,0.06)`. No animation beyond the existing 150ms border-color transition.
- **Preview panel (D-08):** appears below drop zone when a valid ZIP is parsed. Lists exactly 4 rows: `barkeeper.json`, `bar-owner-profile.json`, `inventory.json`, `recipes.json`. Each row shows filename + `.badge-blue` "json" chip + size in bytes (0.82rem text-muted) + ✓ check (green) if present in the ZIP, ✕ (red) if missing.
- **Confirm gate:** if any of the 4 files is missing, "Confirm Import" is disabled and 0.82rem red helper text reads `ZIP must contain all 4 data files.`
- **In-flight import:** "Confirm Import" disabled, label `Importing…`, row-by-row status updates (`Writing barkeeper.json…` → `Writing recipes.json…`). All 4 writes sequential — preserves SHA tracking (Phase 2 key decision).
- **Success:** preview panel collapses, green toast `Import complete.`, then `State.loadAll()` and re-render Settings.

### Anthropic API Key (D-14, aligned with Phase 2 SETTINGS-02 pattern)

- **Placement:** NEW fourth `.settings-section` titled "AI Integration" (GitHub and Anthropic are unrelated services — keep them visually separate).
- **Section heading:** `AI Integration` (1rem amber, `.settings-section__heading`).
- **Helper text:** 0.9rem text-dim: `Your Anthropic API key enables the Generate with AI feature on new recipes. Stored only in this browser.`
- **Input:** `<input type="password" id="st-anthropic-key" autocomplete="off" placeholder="sk-ant-…">`.
- **Show/hide toggle:** `.btn-icon` adjacent. Label text `Show` / `Hide`.
- **Save button:** `.btn .btn-primary .btn-sm` labeled `Save API key`.
- **Storage key:** `bb_anthropic_key`.
- **Status text:** `Saved ✓` (green) or `Cleared` (text-muted) if the input was emptied.

## Anti-Patterns the Checker Must Reject

These patterns would break the contract above. Flag any of them on review.

1. **New CSS variables** added to `:root` for Phase 3 — REJECTED. Reuse `--bg`, `--bg2`, `--bg3`, `--bg4`, `--amber`, `--amber-dim`, `--amber-glow`, `--red`, `--green`, `--blue`, `--border`, `--border2`, `--text`, `--text-dim`, `--text-muted` only.
2. **New font-family declarations** — REJECTED. Inherit `'Georgia', serif` site-wide; monospace via existing classes only.
3. **New font sizes outside the 0.75 / 0.78 / 0.82 / 0.85 / 0.88 / 0.9 / 0.95 / 1 / 1.05 / 1.1 / 1.3 / 1.5 / 1.7 rem scale** — REJECTED.
4. **Modal dialogs for the recipe form** — REJECTED. CONTEXT D-01 mandates in-place content replacement.
5. **New hash routes** (`#recipes/new`, `#recipes/edit`, `#settings/export`) — REJECTED. CONTEXT D-01 and D-10 mandate no new routes.
6. **Per-section selective import checkboxes** — REJECTED. CONTEXT D-08 supersedes EXPORT-04.
7. **`window.confirm()` / `window.alert()` / `window.prompt()` for new UX** — REJECTED. Use existing `.confirm-dialog-overlay` pattern, inline preview panels, or `Utils.toast` for status. (The pre-existing `confirm()` at recipes.js:258 is out of scope.)
8. **Icon library imports** (Lucide, Feather, Heroicons, etc.) — REJECTED. Inline SVG or Unicode glyphs only.
9. **Component libraries** (React, Vue, Alpine, Lit, htmx, etc.) — REJECTED per CLAUDE.md.
10. **CSS-in-JS, Tailwind, styled-components, CSS modules** — REJECTED. All styles append to `app/css/app.css`.
11. **New color hex values** (e.g. a different green, a different amber tint) — REJECTED. Only `rgba()` variations of the four declared accents (amber, green, red, blue) are permitted for tinted backgrounds, following existing patterns at `.menu-item--featured` (line 364) and `.badge-amber` (line 559).
12. **Amber on more than 10% of pixels** at any default render. The Generate button uses `.btn-ghost` precisely to preserve this budget.
