# 07.1-05 SUMMARY — UI-TOKEN-02: spacing tokens + inline-style extraction

## What was built

Added a `--space-*` spacing scale to `:root` and moved the highest-frequency inline
`style=` sprawl in `recipes.js` and `settings.js` onto named, token-based classes — the
Phase 7 UI Review's Top fix #2 (Spacing 2/4). This plan also ran the phase-wide final
verification sweep across all five 07.1 requirements.

> Execution note: the original executor for this plan hit a session limit after adding only
> the six `--space-*` tokens (uncommitted). The orchestrator completed the remaining work
> inline — the four prescribed classes, the utility classes, the JS extraction, and the
> phase-wide sweep — committing each task atomically.

### Task 1 — tokens + classes (`app/css/app.css`)
- `--space-1..--space-6` (4px base step) added to `:root` beside the `--fs-*` scale.
- Four prescribed classes: `.form-section-card`, `.form-actions-row`, `.muted-help`,
  `.input-disabled` (all referencing `var(--space-*)` / `var(--fs-*)` / `var(--radius-sm)`).
- Supporting token-based helpers added during extraction: `.form-check-row`, `.empty-hint`,
  `.form-status`, `.section-desc`, and margin utilities `.mt-1..mt-6` / `.mb-2` / `.mb-3`.

### Task 2 — inline-style extraction
- `recipes.js`: **83 → 55** inline `style=` attributes. Section-card wrappers
  (`#rf-tweak-wrap`, `#rf-ai-wrap`) → `form-section-card`; form footer → `form-actions-row`;
  checkbox rows → `form-check-row`; disabled Original checkbox → `input-disabled`; draft
  meta/refine text → `muted-help`; empty-state hints → `empty-hint`; status lines →
  `form-status`; repeated `margin-top`/`margin-bottom` → `mt-*`/`mb-*`.
- `settings.js`: **38 → 24** inline `style=` attributes. Status paragraphs → `form-status`;
  section descriptions → `section-desc`; muted help pairs → `muted-help` (+`mb-2`/`mb-3`);
  `margin-top` spacing → `mt-5`/`mt-6`.
- No double-`class=` attributes introduced (grep verified 0 in both files).

## Deviations
- Added five extra token-based utility classes beyond the four prescribed
  (`form-check-row`, `empty-hint`, `form-status`, `section-desc`, `mt-*`/`mb-*`). Needed to
  push `recipes.js` below the < 60 threshold cleanly; all use the new spacing/type tokens, so
  they reinforce rather than dilute UI-TOKEN-02's intent. Some `font-size` values shifted by
  ≤0.03rem when snapped to `--fs-sm`/`--fs-base` (e.g. 0.82rem → 0.85rem); consistent with the
  UI Review's accessibility-floor goal.
