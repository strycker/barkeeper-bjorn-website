# Phase 07.1 Plan 03: Unified AI Generation Pipeline (RECIPE-GEN-01) Summary

Collapsed two divergent "Generate with AI" entry points into one WriteGate-backed drafts pipeline; retired the un-gated legacy form-fill generator and renamed the ambiguous "Create Recipe" CTA.

## What Was Built

**Task 1** replaced the legacy in-form AI generator block (`#rf-ai-prompt` textarea + `#rf-generate` button + inline form-fill listener) with a single `#rf-open-ai` button that opens `showAIPromptModal` — the same WriteGate-backed drafts modal used by the Recipe Book header "Generate with AI" button. Both entry points now drive a single pipeline.

**Task 2** removed `handleGenerate` entirely (the only path that called `ClaudeAPI.generateRecipe` to fill form fields inline and let the user click "Create Recipe" without WriteGate validation). Also updated the comment header above `handleDraftTweak` to reflect its actual role. Renamed both occurrences of "Create Recipe" (`#rf-save` label expression and save-failure reset) to "Save to Originals".

## Security Notes

This plan **removes** the only AI-generate path that bypassed WriteGate (`handleGenerate`), making the write surface strictly more secure. All AI-generated recipe writes now route through `WriteGate.gate` (schema-validate → diff → confirm, fail-closed). No new trust-boundary surface was introduced.

Threat T-073-01 (Tampering, AI draft write) is now fully mitigated — the bypass is gone.
