# Phase 7.1 Learnings: Recipes & UI Consistency Cleanup

## Decisions

### Decimal phase insertion instead of renumbering
The cleanup mini-phase was inserted as **Phase 7.1** (GSD decimal insertion) rather than renumbering Portability/Backend/Community from 8/9/10.

**Rationale:** Preserves every downstream phase number and cross-reference; a decimal slots cleanly between Phase 7 and Phase 8, which is exactly "before Phase 8" as requested.
**Source:** STATE.md Roadmap Evolution; ROADMAP.md Phase 7.1

### Streamlined plan path — skip research/UI-SPEC, reuse the Phase 7 UI Review as the design contract
Planned with no CONTEXT.md / RESEARCH.md / UI-SPEC.md.

**Rationale:** This is a refactor of the team's own recently-shipped code; the Phase 7 UI Review already prescribed the exact tokens, six-step scale, and class names, so a formal UI-SPEC would restate it. Kept overhead proportional to a cleanup phase.
**Source:** plan-phase routing decision; 07.1-04/05 plan notes

### Sequential-on-main execution instead of parallel worktrees
Executor subagents ran one at a time on the branch, orchestrator owning STATE.md/ROADMAP.md.

**Rationale:** Parallel commits to one branch race on the git index, and the worktree merge-back machinery is fragile in an ephemeral container. Plans were collision-free across waves, so serialization cost only wall-clock, not correctness.
**Source:** execute-phase orchestration; STATE.md

### Redirect the legacy generator onto the WriteGate drafts pipeline (not "share a core")
RECIPE-GEN-01 removed `handleGenerate` and pointed the in-form "Generate with AI" button at the existing `showAIPromptModal`/`runAIDesign` flow.

**Rationale:** The redirect is the smaller change and converges both surfaces on the already-shipped, WriteGate-gated pipeline — one pipeline, two entry points.
**Source:** 07.1-03-PLAN.md

### Add utility classes beyond the four prescribed to clear the inline-style thresholds
Introduced `empty-hint`, `form-status`, `section-desc`, `form-check-row`, and `mt-*/mb-*` on top of the four UI-Review-named classes.

**Rationale:** The four prescribed classes alone left recipes.js at 68 (target <60); the extras all reference `--space-*`/`--fs-*`, reinforcing UI-TOKEN-02 rather than diluting it.
**Source:** 07.1-05-SUMMARY.md

## Lessons

### Verify the target location against source before planning — the shim was in state.js, not normalize.js
Both the brief and ROADMAP said the compat shim lived in `normalize.js`; it was actually in `state.js` (`_withRecipesShim`/`_resolveSeededForShim` + the `get('recipes')` branch).

**Context:** The planner caught this only by reading the source first; the ROADMAP text and CHIP-03 requirement were then corrected. Planning from the doc alone would have targeted the wrong file.
**Source:** 07.1-01-PLAN.md reality note; plan-checker

### Enumerate ALL call sites of a thing you're deleting — not a representative one
The first plan draft migrated only 1 of dashboard.js's 3 shim reads (missed lines 89 and 185).

**Context:** A shim deletion that leaves a live reader silently returns undefined. The plan-checker (H-1) caught it; the fix was to enumerate every reader (recommender ×9, dashboard ×3, profile, export, claude-api incl. `made_log`) before the deletion task.
**Source:** plan-check H-1

### Acceptance greps must cover every variant or a survivor passes silently
The claude-api.js acceptance grep omitted `made_log`, so a surviving `recipes.made_log` read would have passed the check.

**Context:** When a verification grep enumerates keys, missing one creates a false-green. Fix: include the full alternation `(originals|confirmed_favorites|wishlist|made_log)`.
**Source:** plan-check H-2

### "Substantially reduce inline styles" is harder than it looks — most spacing sits on already-classed elements
The four prescribed extractions only removed ~9 of 83 inline styles in recipes.js; the frequent `margin-top:*` patterns were on elements that already had a `class=`, so they needed class-append (merge), not attribute-replace.

**Context:** Hitting the <60 threshold required token-based `mt-*/mb-*` utilities applied via a generic `class="X" style="P"` → `class="X util"` regex, plus clean text/help extractions.
**Source:** 07.1-05-SUMMARY.md; execution

### A "streamlined" phase still needs a retroactive validation/UAT pass to be truly closed
Skipping RESEARCH/VALIDATION up front left Nyquist gaps; closing the phase required reconstructing VALIDATION.md and adding a CHIP-03 unit test afterward.

**Context:** Streamlining the *plan* is fine, but the close-out (verify-work → validate-phase → extract-learnings) is where the missing artifacts get filled. Budget for it.
**Source:** 07.1-VALIDATION.md

## Patterns

### Headless-Chromium fixture-injection UAT for an auth-gated SPA
Serve the app, set the auth-config localStorage keys so `isConfigured()` is true, `State.set()` the repo's own `data/*.json` (fetched over http), then call `View.render(container)` directly — bypassing the router's auth gate — and assert computed styles + screenshot.

**When to use:** Any GitHub/Supabase-backed SPA whose data views need auth but whose rendering is auth-independent. Gives real visual/runtime evidence without live credentials.
**Source:** 07.1-UAT.md

### Pool-contract regression test via `vm.runInThisContext`
Load `normalize.js` + `state.js` into the node:test harness with stubbed `window`/`localStorage`/`document`, then assert `State.set` → `State.get` produces the canonical pool with no synthesized legacy getters and a stable reference.

**When to use:** Guarding a schema/shape contract or the removal of a compat layer in a no-build browser codebase.
**Source:** tests/phase-07_1-cleanup.test.js; 07.1-VALIDATION.md

### Orchestrator owns tracking; executors commit code + SUMMARY only
Each executor was told explicitly not to touch STATE.md/ROADMAP.md; the orchestrator updated `roadmap update-plan-progress` and STATE after each wave.

**When to use:** Sequential multi-agent execution on one branch, to avoid noisy/conflicting tracking edits and keep STATE prose coherent.
**Source:** execute-phase orchestration

### Generic attribute-merge regex for inline-style → class extraction
`class="([^"]*)" style="PATTERN"` → `class="$1 util"`, plus a standalone `style="PATTERN"` → `class="util"` pass afterward, catches both classed and unclassed elements without hand-editing each.

**When to use:** Bulk-extracting a repeated inline style onto a utility class across a large view file.
**Source:** 07.1-05-SUMMARY.md

### Adversarial plan-check before execution pays for itself
The gsd-plan-checker read the actual source and found 2 blockers (wrong-file assumption's downstream effects) the plan text hid.

**When to use:** Any phase that deletes/migrates shared infrastructure — verify plans against source, not just against the brief.
**Source:** plan-check (2 HIGH + 3 MEDIUM + 2 LOW)

## Surprises

### An executor hit a usage/session limit mid-plan; the orchestrator finished it inline
The 07.1-05 executor died after adding only the `--space-*` tokens (uncommitted). Because nothing was lost (partial working-tree change), the orchestrator completed the four classes, the extraction, and the sweep inline.

**Impact:** No rework; the phase still completed. Reinforces committing atomically and keeping partial state recoverable. The inline completion was noted transparently in the SUMMARY.
**Source:** 07.1-05-SUMMARY.md

### Plans flagged `autonomous: true` still emitted human-verify checkpoints at runtime
The plan-index reported every plan `autonomous: true`, yet 07.1-02/03/04 returned in-task human-verify visual checkpoints during execution.

**Impact:** The frontmatter `autonomous` flag governs the wave gate, not the presence of `checkpoint:` tasks inside a plan. In yolo/headless these auto-approved; worth knowing they aren't the same signal.
**Source:** execute-phase (07.1-04 CHECKPOINT REACHED)

### The app's auth gate lives only in the router, so injected fixtures render full authed views
Setting config + `State.set` fixtures and calling `RecipesView.render` produced a fully-populated Originals tab (chips, overlay actions) with zero live GitHub calls.

**Impact:** Made a genuine headless visual UAT possible in-container — the biggest reason the UI checkpoints could be auto-verified rather than deferred.
**Source:** 07.1-UAT.md

### A cleanup phase turned out to be a net security improvement
Removing `handleGenerate` eliminated the only AI-generation write path that bypassed WriteGate.

**Impact:** ASVS L1 audit closed 11/11 threats with the one "mitigate" being a strengthened boundary, not new work — a refactor that reduced attack surface.
**Source:** 07.1-SECURITY.md; 07.1-03-PLAN.md
