<objective>
Collapse the two divergent "Generate with AI" entry points into one pipeline (RECIPE-GEN-01 /
BL-4), and rename the ambiguous "Create Recipe" CTA.

Today there are TWO generators (verified in source):
  A) LEGACY New-Recipe-form generator: the `#rf-ai-prompt` + `#rf-generate` block in
     renderForm (recipes.js ~1537-1555) → handleGenerate (~1391) → ClaudeAPI.generateRecipe
     → fills form fields inline → user clicks "Create Recipe" (#rf-save label at ~1646).
     This path does NOT go through WriteGate and does NOT create a draft — it writes a
     new Original directly on save.
  B) DRAFTS-tab generator: the Recipe Book header "✨ Generate with AI" button (#rb-generate-ai,
     ~140) → showAIPromptModal (~178) → runAIDesign (~260) → ClaudeAPI.requestJSON(schemaKey:
     'drafts') → WriteGate.gate auto-saves a status:'draft' pool entry → refine card.

Converge on (B), the gated pipeline, per the ROADMAP RECIPE-GEN-01 guidance ("redirect the
legacy entry to the drafts flow"). The legacy form block becomes a redirect into
showAIPromptModal; handleGenerate's un-gated form-fill-then-save flow is retired.

Also rename the new-recipe save CTA from "Create Recipe" to "Save to Originals" (the
ROADMAP's suggested label) so the button reflects the actual outcome.

Purpose: One generation core, one write path (every AI write through WriteGate, fail-closed),
no ambiguous CTA. Sets a clean single pipeline for Phase 8.

REALITY NOTE: handleGenerate (and its helper bindings at ~1655-1664) is the legacy path.
The manual "+ New Recipe" form (no AI) and its non-AI Save remain — only the AI-generate
block inside the form is redirected. No formal UI-SPEC.md exists for this phase.

SECURITY: This consolidation makes the generation surface MORE secure (it removes the only
AI-generate path that bypassed WriteGate). Confirm the unified pipeline still routes every
AI write through WriteGate.gate (it does — runAIDesign line ~377).
</objective>
