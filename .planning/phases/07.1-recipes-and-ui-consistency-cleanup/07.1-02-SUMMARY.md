## Rename-Sync Proof

**Architecture reference:**

- `renderForm` edit-save path (`app/js/views/recipes.js` line 1861-1862):
  ```js
  const idx = pool.findIndex(p => p.id === r.id);
  if (idx >= 0) pool[idx] = { ...pool[idx], ...updated, status: 'original', _source: 'user' };
  ```
  This replaces the pool entry in-place by `r.id`, so the new `name` is stored on the canonical pool entry.

- `_lookupFromPool()` (`app/js/views/recipes.js` line 73-78) returns a `recipeById(id)` function used by every tab's `RecipeChip.bindActions` call (lines 589, 819, 873, 926) to resolve a chip's recipe object from the pool at click time.

- `render()` (`app/js/views/recipes.js` line 80-88) re-derives all filter views — originals, favorites, wishlist, made, drafts — from `State.get('recipes').pool` on every call. After a save, the next render reads the updated name from the pool entry, so Favorites/Wishlist/Made/Recommender chips all show the new name.

No additional code is required for rename-sync.
