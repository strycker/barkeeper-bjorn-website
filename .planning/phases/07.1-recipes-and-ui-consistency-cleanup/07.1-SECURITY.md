# Phase 7.1 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.
> Source: `<threat_model>` blocks of plans 07.1-01 .. 07.1-05 + SUMMARY threat flags
> (both flags: "None"). Verified from-artifacts (State B) against the shipped code on
> branch `claude/hopeful-hamilton-eop3ra`.

**Phase security posture:** NET IMPROVEMENT. This consolidation phase removes code paths
rather than adding input surfaces — 07.1-01 deletes the recipes compat shim, and 07.1-03
removes `handleGenerate`, which was the **only AI-generation write path that bypassed
WriteGate**. After this phase every AI write routes through `WriteGate.gate`
(schema-validate → diff → confirm, fail-closed).

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| localStorage / GitHub JSON → State pool | Persisted recipe data deserialized into the in-memory pool | User's own recipe data (normalized on load) |
| UI click → State pool mutation → GitHub write | Overlay toggles (♥/☆/✓) flip booleans on the user's pool entry, persisted via State.save | Boolean overlay flags / made_log |
| User prompt → Anthropic API → JSON draft → State pool | Untrusted model output parsed and persisted as a recipe draft | Model-generated recipe JSON (WriteGate-gated) |
