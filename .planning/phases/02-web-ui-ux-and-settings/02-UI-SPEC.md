## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, chip gaps |
| sm | 8px | Button gap, bottle-grid gap, form row gap |
| md | 16px | Card margin-bottom, section gap, form-group margin, page-header padding-bottom |
| lg | 24px | Main content horizontal padding (24px), wizard question margin |
| xl | 28px | Main content top padding (28px), page-header margin-bottom |
| 2xl | 32px | Setup card top margin, wizard progress margin |
| 3xl | 40px | New-user welcome top margin, setup card margin |

**Phase 2 spacing contract:** Executors apply ONLY the values in the table above when creating or modifying Phase 2 components. All token values are multiples of 4.

**Legacy values (pre-existing, not modified by Phase 2):**

These values exist in `app/css/app.css` from before Phase 2. Executors MUST NOT change them; they are NOT part of the Phase 2 spacing contract and are listed here for reference only.

| Legacy value | Source | Notes |
|-------------|--------|-------|
| 58px | Header height | Not a multiple of 4; pre-existing fixed layout value |
| 20px | Header horizontal padding | Not on the 8-pt grid; pre-existing |
| 6px 12px | Nav item padding | Not a multiple of 4; pre-existing |

Other exceptions:
- Wizard max-width: 540px (layout constraint; not a spacing token)
- Touch targets for "Skip for now" links: minimum 44px tall (accessibility)

> Source: `app/css/app.css` measurements confirmed by codebase grep. 28px is the de facto xl in this codebase; declared as xl above. 32px is 2xl.

## Typography

Four canonical size tokens for Phase 2. All component-level font sizes reference exactly one of these four tokens.

| Token | Size | Weight | Line Height | Usage |
|-------|------|--------|-------------|-------|
| small | 13px (0.88rem) | 400 | 1.4 | Nav links, badge text, helper text, slider pole labels, "Middle" center label, slider labels, progress banner text, avatar caption, settings section headings (uppercase), inventory section titles (uppercase) |
| body | 15px (1rem) | 400 | 1.6 | Default body text, card content, form labels, wizard option text, chip text |
| heading | 20px (1.3rem) | 400 | 1.3 | h2, wizard question headers (`wizard-question`), section titles, h3 treated as heading |
| display | 26px (1.7rem) | 400 | 1.2 | h1, page titles (amber color) |

Font weights used: 400 (regular) and 600 (semibold, used for CTA link text and selected states only). No other weights.

Font family for ALL elements: `'Georgia', serif` — no system-ui, no sans-serif in this phase.

**Token mapping for Phase 2 component additions (previously declared as separate sizes, now mapped):**

| Component use | Previous size | Canonical token |
|--------------|--------------|----------------|
| Progress banner text | 14px | small (13px) |
| Onboarding wizard question | 19px | heading (20px) |
| "Coming soon" badge text | 11px | small (13px) |
| Inventory section title | 12px | small (13px) |
| Slider pole labels | 13px | small (13px) |
| Slider center label "Middle" | 11px | small (13px) |
| h3 (e.g. settings sub-headings) | 16px | heading (20px) |

Additional typographic modifiers that may be combined with size tokens (do not create new size tokens):
- `text-transform: uppercase` + `letter-spacing: 0.1em` → settings section headings (small token)
- `text-transform: uppercase` + `letter-spacing: 0.12em` → inventory section titles (small token)
- `letter-spacing: 0.08em` → "Coming soon" badge (small token)
- `font-style: italic` → avatar caption (small token)
- `font-weight: 600` → CTA link text in progress banner (small token)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f0f` (`--bg`) | Page background, main surface |
| Secondary (30%) | `#1a1a1a` (`--bg2`) | Cards, header, nav, wizard card, settings sections |
| Tertiary surface | `#242424` (`--bg3`) | Hover states, inputs, chips, slider track background |
| Input background | `#2e2e2e` (`--bg4`) | Input fields, muted containers |
| Accent (10%) | `#d4943a` (`--amber`) | See reserved-for list below |
| Destructive | `#e05555` (`--red`) | "Reset all data" button, chip remove on hover, logout confirm |
| Success | `#4caf7d` (`--green`) | Save confirmations, toast success |
| Informational | `#5b9bd5` (`--blue`) | Slider track left-end gradient only |
| Border primary | `#333` (`--border`) | Card borders, section dividers |
| Border secondary | `#444` (`--border2`) | Input borders, focused-adjacent elements |

**Accent (`--amber`) reserved for:**
1. Page h1 headings
2. Active nav link text
3. Header logo text
4. Primary CTA buttons (`.btn-primary` background)
5. Wizard progress bar fill
6. Selected `.wizard-option` border and text
7. Stat values in stats bar
8. Onboarding welcome avatar border/glow (new: 2px solid `--amber` ring, 8px border-radius)
9. Slider thumb border (existing `.axis-slider` thumb)
10. Settings section heading ("Bartender Identity", "GitHub Connection", etc.)
11. Featured dashboard quick-action card border (`--amber-dim`)

**Accent NOT used for:**
- Body text, labels, descriptions
- Non-featured dashboard cards
- Progress banner background (use `--bg2` with `--amber` text link only)
- "Coming soon" badge background (use `--bg4`)

**Phase 2 new semantic uses:**
- Danger Zone card: `--red` border (1px solid), `--red` heading text
- "Coming soon" grayed card: opacity 0.45 applied to entire `.menu-item`; pointer-events none; badge background `--bg4`, text `--text-muted`
- Progress banner: background `--bg3`, border-left 3px solid `--amber`, text `--text-dim`, CTA link `--amber`

**Focal points:**
- Dashboard: the featured "What Can I Make Right Now?" card (`.menu-item--featured`) is the primary visual anchor — amber border distinguishes it from all other grid items.
- Onboarding welcome screen (step 1): the Bjorn avatar (120×120px, amber ring) is the primary visual anchor — it is the only circular amber-bordered element on the screen.

> Source: `app/css/app.css` `:root` confirmed. Color roles from CONTEXT.md locked decisions + existing CSS patterns.

### 1. Flavor Axis Slider (new — used in Onboarding steps 10–15)

Extends the existing `.axis-slider` CSS class from the Profile view.

**Structure:**
```
.axis-slider-group
  .axis-slider-poles          ← flex row: left-label | spacer | right-label
    .axis-pole-label--left    ← small token (13px), --text-dim
    .axis-pole-label--right   ← small token (13px), --text-dim, text-align right
  input[type=range].axis-slider  ← existing class; width 100%
  .axis-slider-center-label   ← "Middle", small token (13px), --text-muted, text-align center, margin-top 4px
```

**Behavior:**
- Value range: 0.0 to 1.0 (step 0.01)
- Default / skip default: 0.5
- Center tick visible at 50% position via `background: linear-gradient(...)` existing style
- "Middle" text label is always visible below the track center (static position, not dynamic)
- On skip: writes `{ value: 0.5, _skipped: true }` to profile; slider thumb snaps to center
- On save: writes float value 0.0–1.0

**Pole label pairs (locked by CONTEXT.md — do not alter):**

| Axis | Left label | Right label |
|------|-----------|-------------|
| sweetness | Dry / bone-dry | Sweet / balanced |
| acid | Smooth / low-acid | Bright / tart |
| strength | Light / low-ABV | Strong / high-proof |
| complexity | Simple / clean | Complex / layered |
| season | Bright / summery | Warm / wintry |
| risk | Familiar / classic | Adventurous / unusual |

### 2. Free-Text Inventory Paste (new — Onboarding step 9)

**Structure:**
```
.wizard-paste-wrap
  label                       ← "Paste your spirits list (comma-separated)"
  textarea.wizard-paste-input ← 5 rows, placeholder: "e.g. Bulleit Bourbon, Aperol, Angostura Bitters"
  .wizard-paste-hint          ← small token (13px), --text-muted: "You can add bitters, syrups, and other pantry items from the Inventory view."
  .wizard-chip-preview        ← flex-wrap, gap 8px; appears after parsing
    .bottle-chip (existing)   ← each parsed item; includes chip-remove button
  button.btn.btn-primary      ← "Looks good →" (hidden until ≥1 chip exists)
```

**Behavior:**
- Parse on blur or on a "Preview" button click (not on every keystroke)
- Split on comma, trim whitespace, drop empty strings
- Each chip shows the trimmed bottle name + remove button (×)
- "Looks good →" writes all remaining chips to `inventory.spirits` as strings
- Skip is available: "Skip for now →" link below the textarea; saves nothing, advances step

## Accessibility Requirements

- All images must have descriptive `alt` attributes: avatar images use alt="[barkeeper name]"; hero image uses alt=""  (decorative)
- "Skip for now →" links must be keyboard-focusable (not `<span>` — use `<button type="button">` styled as link)
- Slider inputs must have `aria-label="[Axis name] preference"` and `aria-valuemin="0"` `aria-valuemax="1"` `aria-valuenow="{value}"`
- "Coming soon" disabled cards: add `aria-disabled="true"` and `tabindex="-1"`; ensure the toast fires on Enter keypress on the parent grid (keyboard accessible)
- Settings Danger Zone: `role="region"` `aria-label="Danger Zone"` on `.settings-section--danger`
- Logout and Reset confirmation: dialog must trap focus (use native `confirm()` in Phase 2; full focus trap deferred)
- Color is never the sole differentiator — disabled cards use opacity AND badge text
- Minimum touch target for all buttons: 44px height
