## Spacing Scale

All existing spacing follows a loose 4-point base. Phase 4 adds no new scale values ‚Äî extend existing patterns only.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, tier dot margin, chip internal padding-y |
| sm | 8px | Chip grid gap, button row gaps, form row gaps |
| md | 16px | Section padding, form group margin, card padding |
| lg | 24px | Section bottom margin (`margin-bottom: 28px` rounds to this zone), tab content padding |
| xl | 32px | Page header bottom margin |
| 2xl | 48px | Recommender mood sidebar top offset on desktop |
| 3xl | 64px | Not used in Phase 4 |

Exceptions:
- Sticky save bar top offset: `top: 60px` (matches header height of 58px + 2px border ‚Äî existing pattern from inventory.js)
- Inline edit form left border accent: 3px solid `--amber` (same as `.dash-progress-banner` left border pattern)
- Touch-target minimum for strainer checkboxes: 44px tall rows (accessibility requirement for tap targets)
- Mood slider thumb size: 16px diameter (matches existing `.axis-slider::-webkit-slider-thumb`)

## Typography

Existing type scale is honored exactly. Phase 4 introduces no new sizes or weights. The scale is exactly 4 sizes.

| Role | Size | Weight | Line Height | CSS |
|------|------|--------|-------------|-----|
| Small / muted / label / meta | 12px (0.78‚Äì0.82rem) | 400 | 1.4 | chip text, card meta, form hints, section titles, tier labels, score labels |
| Body | 15px | 400 (normal) | 1.6 | `html, body` base |
| Heading | 20px (1.3rem) | 400 | 1.2 | `h2` ‚Äî section headings |

Font weights used: 400 (regular) and 600 (bold/semibold ‚Äî `.btn-primary`, `.settings-section__heading`, headings where `font-weight: bold`). No third weight is introduced.

Specific Phase 4 type usages:
- Inline edit form field labels: 12px / 400 / `color: var(--text-dim)`

## Component Contracts

### 1. Bottle Chip (Extended ‚Äî D-01, D-07)

**Existing chip** (`bottle-chip`): tier dot + style name + remove button.

**Phase 4 chip display:**
- Primary text: `bottle.style` (e.g. "Bourbon") at 15px / `--text`
- Secondary text: `bottle.brand` when present ‚Äî 12px / `--text-muted` ‚Äî rendered inline after a middot separator: `¬∑ Maker's Mark`
- Tier dot: 7px circle, color per tier palette above
- Click target (not remove button): entire chip surface minus the √ó button triggers inline edit expand (D-07)
- Selected/active chip state: `border-color: var(--amber)` and `background: rgba(212,148,58,0.08)` ‚Äî same as `.wizard-option.selected`

**Chip tooltip (HTML `title` attr):** Full bottle object summary: `"{style} ‚Äî {brand} ‚Äî {tier} tier"`, each field omitted if null.

### 2. Inline Edit Form (D-07, D-08, D-09)

**Position:** Renders as a block element immediately below the `.bottle-grid` of the section containing the clicked chip. Not a modal ‚Äî no overlay, no z-index stacking.

**Visual anchor:** Left border `3px solid var(--amber)` ‚Äî same treatment as `.dash-progress-banner`. Background: `var(--bg2)`. Border: `1px solid var(--border2)`. Border-radius: `var(--radius)` (8px). Padding: 16px.

**Default visible fields (D-08):**
- `style` ‚Äî text input, required, focus-amber border
- `type` ‚Äî combobox/select with standard type options + "Add new category" option at bottom

**Expanded fields (revealed via "More fields" toggle):**
- `brand` ‚Äî text input
- `tier` ‚Äî select (6 options from new tier system)
- `best_for` ‚Äî select: Sipping / Mixing / Both
- `notes` ‚Äî textarea, 2 rows max height, resizable vertically

**Expand toggle:** Text link "More fields ‚ñæ" / "Fewer fields ‚ñ¥" in `--text-dim` at 12px. No animation required; toggle show/hide is instant (no transition).

**Buttons:**
- Save: `.btn.btn-primary.btn-sm` ‚Äî "Save Bottle" ‚Äî patches State, marks dirty, collapses form
- Revert: `.btn.btn-ghost.btn-sm` ‚Äî "Revert Changes" ‚Äî restores snapshot values, collapses form
- Buttons sit in a `display:flex; gap:8px; justify-content:flex-end` row at bottom of form

**Form collapse:** On Save Bottle or Revert Changes, the form is removed from DOM. The chip grid re-renders with updated values.

**One-at-a-time rule:** Clicking a second chip while a form is open closes the open form first (revert behavior, no save prompt) then opens the new form below the new chip's section grid.

### 3. Add-Bottle Row (Extended ‚Äî D-01, D-02, D-24)

**Existing:** Single text input + tier select + "Add Bottle" button.

**Phase 4 change:** Tier select is removed from the add-bottle row (tier is set in the edit form after adding). The add-bottle row becomes: style/name input + "Add Bottle" button only. Tier defaults to null on add.

**Canonical name banner (D-24):** Renders as a block element immediately below the add-bottle input (not inside the input, not a tooltip). Visible only when `CanonicalNames.suggest(value)` returns a result.

Banner markup pattern (new CSS class `.canonical-suggestion`):
- Background: `rgba(212,148,58,0.08)` ‚Äî same amber tint as selected wizard option
- Border: `1px solid var(--amber-dim)`
- Border-radius: `var(--radius-sm)` (5px)
- Padding: `6px 10px`
- Font: 12px / `--text`
- Text: "Did you mean: **{canonical name}**?"
- Action: inline `<button>` labeled "Use it" ‚Äî `.btn-ghost.btn-sm` style, displayed inline (not block)
- Behavior: clicking "Use it" sets input value to canonical name and removes banner; banner clears on input clear

### 4. Equipment Tab (D-11, D-12)

**Tab label:** "Equipment" ‚Äî added as 4th tab after "Vetoes & Substitutes" in the existing `.tabs` tab row.

**Strainer multi-select checkbox grid:**
- Container: `display: grid; grid-template-columns: repeat(2, 1fr); gap: 8px;` ‚Äî 2-column grid on desktop, 1-column on mobile (‚â§480px)
- Each option: `<label>` row with checkbox + text ‚Äî minimum 44px height for touch targets
- Checkbox: native `<input type="checkbox">` styled via existing form conventions; no custom checkbox component
- Options (fixed, non-extensible): Hawthorne, Julep, Fine Mesh, Conical
- Checked state: label text color changes to `var(--amber)`, border-color `var(--amber)`
- Data path: `inventory.equipment.strainers` array

**Empty state for Equipment tab:** "No equipment tracked yet. Check the strainers you own below." in `--text-dim` at 15px.

### 5. Recommender Layout (D-20, D-21)

**Desktop layout (‚â• 860px):** Two-column CSS grid:
- Left column: `280px` fixed width ‚Äî contains mood sliders, scope toggle, base-spirit filter chips, occasion filter chips
- Right column: `1fr` ‚Äî contains recipe card list
- Gap: `24px`
- CSS: `display: grid; grid-template-columns: 280px 1fr; gap: 24px; align-items: start;`

**Mobile layout (< 860px):** Single column. Controls stack above results. Mood sliders are collapsed behind "Adjust Mood" toggle (D-21).

**"Adjust Mood" toggle (mobile only):**
- Button: `.btn.btn-secondary.btn-sm` labeled "Adjust Mood ‚ñæ" / "Hide Sliders ‚ñ¥"
- Sliders reveal below the button with no animation (instant show/hide)
- Toggle state is not persisted

### 6. Mood Sliders (D-13, D-14, D-15, D-16)

**6 axes** (matching existing `bar-owner-profile.json` flavor profile axes):
1. Sweetness (Dry ‚Üê ‚Üí Sweet)
2. Acidity (Low ‚Üê ‚Üí High)
3. Strength (Light ‚Üê ‚Üí Strong)
4. Complexity (Simple ‚Üê ‚Üí Complex)
5. Season (Summer ‚Üê ‚Üí Winter)
6. Risk (Safe ‚Üê ‚Üí Adventurous)

**Visual:** Each slider uses the existing `.axis-slider` CSS class (webkit-styled range input, 4px track, 16px thumb, amber border). Pole labels use `.axis-pole-label--left` / `.axis-pole-label--right` at 12px / `--text-muted`. Axis name label at 12px / `--text-dim` above the slider.

**Section heading:** "Mood" in `.inventory-section-title` style (12px, uppercase, letter-spaced, `--amber-dim`).

**Event:** `onchange` only ‚Äî not `oninput`. Results re-rank on slider release.

**"Save to profile" button:** `.btn.btn-primary.btn-sm` ‚Äî label "Save to Profile" ‚Äî positioned at bottom of the mood slider panel. Calls `State.save('profile')` and shows `Utils.showToast('Profile updated ‚úì')`.

**Reset link:** Small text link "Reset to saved" at 12px / `--amber-dim` next to the Save button. Resets all slider values to the last-saved profile values without calling State.save.

### 7. Scope Toggle (D-17, D-18)

**Control type:** Segmented button group ‚Äî 3 options displayed as `.rec-filter-chip`-style buttons in a single row:
- "Only what I have"
- "Allow 1 missing"
- "Allow 2 missing"

Active state: `background: var(--amber-dim); border-color: var(--amber); color: #fff;` ‚Äî same as `.rec-filter-chip.active`.

**Result sections (cumulative):**
- Level 0 ("Only what I have"): one section ‚Äî "You Can Make These" ‚Äî `N` count badge
- Level 1 ("Allow 1 missing"): adds section below ‚Äî "One Bottle Away" with amber left border heading
- Level 2 ("Allow 2 missing"): adds section below that ‚Äî "Two Bottles Away" with `--blue` left border heading

**Section headings:** `h3` style ‚Äî 17px / `--text` / normal weight, with count badge (`.rec-badge` pattern).

**Two-away card additions (D-18):** Below the ingredient chip row, two missing-ingredient rows are shown. Each row:
- Text: missing ingredient name in `--amber`
- Inline link: "Add to shopping list ‚Üí" in `--blue` / 12px ‚Äî appends item to `inventory.unassigned` shopping list and shows toast "Added to shopping list"

### 8. Occasion Filter Chips (D-19)

**Position:** Below scope toggle, above recipe card list (in sidebar on desktop, stacked on mobile).

**Pattern:** Same `.rec-filter-chip` / `.rec-filter-chip.active` pattern as existing base-spirit filter.

**Label:** "Occasions" ‚Äî section label at 12px / `--amber-dim` above the chip row.

**Multi-select behavior:** Multiple occasion chips can be active simultaneously. Results are filtered to recipes matching ANY selected occasion (OR logic). When no chips are active, all results shown.

**Chip values:** Derived at runtime from the classics DB `recipe.occasion` strings. De-duplicated and sorted alphabetically.

**Empty state:** If no occasion data exists in DB, occasion section is not rendered (graceful degradation).

## Interaction States

### Dirty State Flow (Inventory)

All inventory edits follow the existing three-step pattern:
1. User action ‚Üí `State.patch()` on the in-memory object
2. `markDirty()` ‚Üí sticky save bar appears (`display: flex`, `top: 60px`)
3. "Save to GitHub" ‚Üí `State.save('inventory')` ‚Üí save bar hidden, toast shown

Inline edit form Save Bottle button triggers steps 1‚Äì2 only. The sticky save bar handles step 3.

### Migration Write (D-06)

One-time migration on first `State.save('inventory')` after Phase 4 deploy. All old bottle string/object shapes normalized to new `bottleEntry` shape. Old tier values cleared. Sequential `State.save()` pattern from `settings.js:289‚Äì302` must be followed ‚Äî no parallel saves.

### Recommender Re-score

Mood slider `onchange` constructs an ephemeral profile object (spread of saved profile with slider overrides) and passes it to `RecommenderEngine.recommend(inventory, ephemeralProfile)`. The results object is replaced in `_results`. `_rerender()` is called. The page does not reload ‚Äî only `#rec-list` innerHTML is replaced.

### Canonical Name Suggestion (D-22, D-24)

- Trigger: `oninput` on the add-bottle style/name text field
- When `CanonicalNames.suggest(value)` returns a match: render `.canonical-suggestion` banner below input
- When input is cleared or matches exactly: remove banner
- Edit-distance threshold: Levenshtein ‚â§ 2 for inputs ‚â§ 6 chars; ‚â§ 3 for inputs > 6 chars (Claude's Discretion)
