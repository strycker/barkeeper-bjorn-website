<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → library DOM | link title/description/tags and especially the URL are user-entered untrusted strings rendered as HTML/href |
| library.json → State/GitHub | link payloads cross into the user's system of record |
| model output → drawer | lesson/library Ask-Bjorn answers stream into the shared drawer (handled by 07-03) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-17 | Tampering (XSS) | library href / title / tags | mitigate | `_safeHref` allows only http(s) URLs as anchors (no javascript:/data:); all titles/descriptions/tags escaped via `Utils.escapeHtml` before innerHTML (Security V10) |
| T-07-18 | Tampering (XSS) | classroom lesson body | mitigate | Lesson bodies are static, but still rendered through `Utils.escapeHtml` defensively before innerHTML |
| T-07-19 | Tampering | library write payload | mitigate | Writes go through Normalize.library (coerce/drop-unknown) + State.save; no AI payload here, so a standalone write-gate is not required (user-typed, validated by Normalize) |
| T-07-20 | Spoofing (no-key bypass) | classroom/library Ask-Bjorn | mitigate | getKey()==='' check before openDrawer; openDrawer additionally self-gates (07-03) |
| T-07-21 | DoS (load failure) | loadAll on missing library.json | mitigate | Committed seed + tolerant-404 load (Pitfall 1) so existing repos do not break on upgrade |
</threat_model>
