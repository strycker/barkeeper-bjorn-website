<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → api.anthropic.com | BYOK key in x-api-key; streamed model output (untrusted) crosses back into the DOM |
| model output → #chat DOM | streamed tokens rendered into transcript bubbles |
| bb_chat_history → GitHub | manual Save commits the transcript to the user's repo |
| navigation → in-flight stream | leaving #chat / closing drawer must cancel the request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-11 | Tampering (XSS) | _renderTranscript / onText | mitigate | Streamed tokens appended as text nodes; all stored turns escaped via `Utils.escapeHtml` before innerHTML (Pitfall 3 / Security V10) |
| T-07-12 | DoS (user wallet) | streamMessage on nav / drawer close | mitigate | AbortController.abort() in cleanup() and on drawer close (CHAT-06); 429 surfaced via thrown message, no retry loop (CHAT-09 from 07-01) |
| T-07-13 | Info Disclosure | bb_chat_history persistence | accept | Transcript is the user's own conversation in their own browser; key is never part of a turn (only the x-api-key header carries it) — no key in history |
| T-07-14 | Tampering | drawer abort touching persisted thread | mitigate | Drawer uses an in-memory turns array only; cleanup/close never writes bb_chat_history (Pitfall 5); Task 1 grep confirms openDrawer has no bb_chat_history reference |
| T-07-15 | Spoofing (no-key bypass) | CHAT-03 gate | mitigate | render()/openDrawer() check getKey()==='' before any call and show the Settings link (AI-SPEC §6 no-API-key gate) |
| T-07-16 | DoS (cost) | per-turn context size | mitigate | buildContext() cached block (D-05) + windowed messages (summary+recent, D-07) keep per-turn input small; max_tokens cap on every body |
</threat_model>
