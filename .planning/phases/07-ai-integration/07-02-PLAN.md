<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → Settings UI | bb_api_log (recorded model output metadata) and bb_chat_model rendered into the DOM |
| bb_api_log → clipboard | user copies raw log JSON out of the app |
| user → bb_chat_model | selector value drives which model every later call uses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-07 | Info Disclosure | AI-09 log render / copy | mitigate | Panel reads only ts/type/model/usage; Task 2 asserts no key/prompt/system field is read or copied (FM #4); copy emits the same key-free array stored by appendLog |
| T-07-08 | Tampering (XSS) | renderAiLog innerHTML | mitigate | Every interpolated value (type/model/usage/timestamp) wrapped in `Utils.escapeHtml` before innerHTML (Security V10 / Pitfall 3) |
| T-07-09 | Tampering | bb_chat_model value | accept | Selector is constrained to three hardcoded option values; an out-of-range bb_chat_model only causes Anthropic to 400 the call, surfaced via existing error handling — no data-integrity impact |
| T-07-10 | Info Disclosure | Reset all data | mitigate | Task 2 confirms bb_chat_model + bb_api_log are swept so a shared/handed-off browser does not retain AI metadata |
</threat_model>
