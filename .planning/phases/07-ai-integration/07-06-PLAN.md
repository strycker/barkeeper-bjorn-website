<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported MD / pasted lines → model | untrusted user-supplied text sent to Anthropic for parsing |
| model output → State/GitHub | parsed/repaired payloads cross into the user's system of record |
| model output → DOM | diff previews + parsed chips render into the UI |
| localStorage bb_parse_cache → app | cached parse results read back |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-28 | Tampering | AI-08/AI-10 write | mitigate | Every import/repair write goes through `WriteGate.gate` (schema-validate -> diff -> confirm); requestJSON fails closed; invalid payload never written (FM #1/#2) |
| T-07-29 | Tampering (prompt injection) | imported MD content | mitigate | Imported content is treated as untrusted: it is parsed, Normalized, schema-validated, and diff-confirmed before any write — injected instructions cannot bypass the schema gate (Security Domain) |
| T-07-30 | Tampering (XSS) | import diff / parsed chip preview | mitigate | All parsed/model output escaped via `Utils.escapeHtml` before innerHTML (Security V10 / Pitfall 3) |
| T-07-31 | DoS (cost) | AI-11 fallback / AI-08 per-file | mitigate | AI-11 fires only on low confidence + caches by input string (Pitfall 6); Haiku tier; max_tokens caps on every requestJSON; no auto-retry beyond requestJSON's single retry |
| T-07-32 | Info Disclosure | bb_parse_cache | accept | Cache holds only parsed bottle objects keyed by the user's own input; no key is ever cached or logged (FM #4) |
| T-07-33 | Tampering | AI-11 auto-write | mitigate | Paste-a-line stages a reviewable chip; the existing user-review + save path applies — no direct AI->GitHub write |
</threat_model>
