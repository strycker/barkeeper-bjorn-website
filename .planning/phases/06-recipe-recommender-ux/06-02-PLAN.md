<objective>
Implement D-06: when the universal detail modal opens an Original (`_source === 'originals'`),
render name / ingredients (add+remove rows) / method / glassware / garnish as EDITABLE inputs;
for any other `_source` (classics-db, ai-generated, undefined) keep them READ-ONLY. Saving an
Original performs a DUAL-WRITE inside a single `State.patch('recipes', ...)`: update the
canonical `recipes.originals` record AND patch the inline copy in the originating list
(confirmed_favorites / wishlist / made_log).

Purpose: "Pages are organizational views; chips are the interface." An Original reached from
any list edits the same canonical recipe. Non-originals stay locked to prevent editing data
the user does not own.

Output: extended `showRecipeDetail` in recipes.js + modal-input CSS.

DEPENDS ON 06-01: dual-write inline-copy lookup and notes/tally lookups use `Utils.sameRecipe`
(name+base). The editable path only fires for entries whose stored `_source` is `'originals'`
— D-07 (06-03) makes the Recommender write `_source:'originals'` for Originals; until then this
path is reachable for Originals favorited/made via the modal's own made-again write (which
already preserves `recipe._source`).
</objective>

    ```javascript
    State.patch('recipes', r => {
      const canon = (r.originals || []).find(o => o.id === origId)
        || (r.originals || []).find(o => Utils.sameRecipe(o, origProbe));
      if (canon) Object.assign(canon, edited);
      if (listKey) {
        const copy = (r[listKey] || []).find(x => Utils.sameRecipe(x, origProbe));

      }
    ```
